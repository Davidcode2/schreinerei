# Phase 34 Plan

## Goal

Embed the existing fleet calendar into the `/fleet` page while keeping the current fleet tabs and lists usable below it.

## Decisions

- Reuse the existing `CalendarView` implementation instead of creating a second calendar path.
- Keep `/fleet/calendar` available during Phase 34, but make it a thin standalone wrapper around the same embedded-capable component.
- Keep the current resource-card reservation entry points unchanged for now; the booking interaction redesign belongs to Phase 35.
- Add a clear calendar section above the existing fleet management tabs so the page remains readable on mobile.

## Tasks

1. Update `CalendarView` so it can render in embedded mode without the standalone page title or back button.
2. Render the embedded calendar section at the top of `FleetPage`.
3. Keep the standalone `/fleet/calendar` route using the same implementation.
4. Add frontend coverage that verifies the fleet page includes the embedded calendar section above the existing content.

## Success Criteria

- Fleet page shows the reservation calendar directly below the page header.
- Existing fleet tabs and list content remain below the calendar.
- `/fleet` and `/fleet/calendar` use the same calendar implementation.
