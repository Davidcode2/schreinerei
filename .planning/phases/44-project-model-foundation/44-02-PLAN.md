## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| admin browser → project planning UI | Form inputs and assignment actions are user-controlled before they hit admin-only APIs |
| project planning UI → sites API | Client must not bypass existing admin/tenant constraints while broadening fields |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-44-05 | Tampering | project planning form state | mitigate | Constrain `project_type` to explicit frontend options that mirror the backend enum and keep validation for dates/hours before submission. |
| T-44-06 | Elevation of privilege | assignment management UI | mitigate | Reuse existing admin-only assignment endpoints and do not add client-only role editing shortcuts that bypass the backend permission checks. |
| T-44-07 | Information disclosure | user/assignment listings | mitigate | Use the tenant-scoped `useUsers` and `useSiteAssignments` hooks only; do not introduce cross-tenant search or unbounded user fetch logic. |
| T-44-08 | Denial of service | edit surface complexity on mobile | mitigate | Keep the form to essential planning fields per D-04 and the mobile-first checklist so users are not forced through a heavy multi-step workflow. |
</threat_model>
