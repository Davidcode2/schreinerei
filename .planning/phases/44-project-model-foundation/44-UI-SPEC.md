# Phase 44 UI Spec

## Goal

Preserve the current mobile-first Baustelle experience while starting the shift toward project-oriented terminology and broader execution context.

## Required UI Outcomes

1. Existing screens remain usable on phone and tablet without extra steps.
2. Any new terminology introduced in this phase should help users understand the broader project model, not confuse them.
3. If new project-type fields are exposed, they must be simple to choose and understandable in carpentry terms.
4. Planning fields remain editable from the main project detail/create surfaces.

## Constraints

- Do not redesign the full detail experience in this phase.
- Do not add heavy desktop-only planning UIs.
- Prefer additive copy and field changes over multi-step workflow changes.
- Maintain compatibility with existing mobile navigation, camera flow, and active-site/project context patterns.

## Acceptance Lens

- A worker should still recognize the same core flow they already use.
- A manager should better understand that workshop work and Baustellen are both projects in the system.
- No new field should feel like office bureaucracy unless it directly enables later execution/reporting value.
