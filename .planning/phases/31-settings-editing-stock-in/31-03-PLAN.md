<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| detail-page dialogs → inventory API | Untrusted quantity, notes, and location edits are turned into stock and metadata mutations |
| direct-set stock correction UI → adjust endpoint | A user-entered target quantity must be translated safely into the backend's delta-based contract |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-31-09 | Tampering | `MaterialEditDialog.tsx` stock correction | mitigate | Compute `delta = targetQuantity - material.quantity` locally and send it only through `useAdjustMaterialStock` with a fixed reason string, rather than letting the user type arbitrary backend payload keys. |
| T-31-10 | Tampering | location clearing logic | mitigate | Convert blank location input into the exact backend-supported `clear_location: true` contract so the client does not send ambiguous `null`/empty-string combinations. |
| T-31-11 | Repudiation | stock-in flow | mitigate | Use the dedicated `/stock-in` endpoint from Phase 30 instead of reusing withdraw/adjust flows, preserving the backend's `material_added` audit trail. |
| T-31-12 | Denial of service | repeated dialog submits | mitigate | Disable submit buttons while mutations are pending so duplicate stock-in or correction requests cannot be fired by rapid repeated clicks. |
</threat_model>
