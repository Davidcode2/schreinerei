## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser form state → inventory API | Untrusted user input becomes category/material mutation payloads |
| inventory header navigation → protected route | User-controlled route transitions must land on an authenticated in-app page only |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-31-01 | Tampering | `frontend/src/lib/api/hooks/useInventory.ts` | mitigate | Send only the exact DTO keys already defined by backend contracts (`name`, `description`, `location`, `min_quantity`, `clear_location`, `quantity`, `reason`, `notes`) and avoid ad-hoc payload reshaping in page components. |
| T-31-02 | Spoofing | `/settings/inventory` route | accept | Route remains under the existing `AuthGuard` tree in `App.tsx`; no public route is introduced. |
| T-31-03 | Repudiation | stock correction/stock-in actions | mitigate | Keep hooks mapped to backend endpoints that already emit domain events from Phase 30 instead of inventing client-only state changes. |
| T-31-04 | Denial of service | React Query cache invalidation | mitigate | Invalidate only the narrow keys `['categories']`, `['materials']`, `['material']`, and `['low-stock']` instead of clearing the entire cache. |
</threat_model>
