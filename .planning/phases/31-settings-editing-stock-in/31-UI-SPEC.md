## Interaction Contract

- Add a visible gear entrypoint on `InventoryListPage` header that routes to `/settings/inventory`; do not hide the route inside generic settings content.
- The `/settings/inventory` page must use `PageHeader` + stacked cards, matching the existing settings page rhythm (`space-y-6`, centered readable width, card sections).
- Category management uses one list row per category with: name, optional description, edit action, delete action. Edit opens an inline dialog; delete opens `AlertDialog` confirmation.
- When category deletion is blocked by referenced materials, keep the user on the same page, keep the category row visible, and show the fixed conflict copy above or inside the affected row/dialog. Do not collapse it into a generic toast-only failure.
- `InventoryDetailPage` action order: 1) `Material einlagern` as the primary button, 2) `Material entnehmen` as outline button, 3) `Material bearbeiten` as outline button. `Nachbestellen` remains conditional on low stock.
- Material editing uses one dialog for location, minimum quantity, and arbitrary stock correction. Group fields in this order: `Lagerort`, `Mindestbestand`, `Bestand korrigieren`.
- `Bestand korrigieren` is a direct-set action, not a delta action. Show helper text: `Setzt den verfügbaren Bestand direkt auf diesen Wert.`
- Stock-in uses a dedicated dialog, never the withdraw dialog. Required field: quantity. Optional field: notes. Show current stock in a muted summary chip/card at the top.
- On successful stock-in, close the dialog and show toast pattern `„{quantity} {unit} eingelagert“`.
- On successful material edit, close the dialog and show toast pattern `„Material aktualisiert“`.
- On overview cards, show the category name on every material entry directly under the material name as a subdued secondary badge or label before description/location metadata.
