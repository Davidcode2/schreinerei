<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| React activity feed → delete mutation | Untrusted clicks attempt destructive operations and may target stale or unauthorized entries |
| delete mutation → backend route | UI must send exact site/activity IDs but cannot be trusted to enforce authorization |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-37-06 | T/E | delete button visibility | mitigate | Render the affordance only from backend `can_delete`; never infer ownership from frontend auth IDs or card content. |
| T-37-07 | R | destructive confirmation UX | mitigate | Require an explicit confirmation dialog before invoking the mutation so accidental taps do not trigger deletion. |
| T-37-08 | I | media feed card wiring | accept | The UI already displays entry content and media metadata; adding a delete button does not expose new sensitive data. |
| T-37-09 | D | repeated mutation clicks | mitigate | Disable the confirm action while the mutation is pending and close/reset dialog state only after completion. |
</threat_model>
