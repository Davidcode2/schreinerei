# Phase 41 Plan

## Goal

Expose a `projects` boundary in code without destabilizing the current `sites` runtime model.

## Decisions

- Keep the current `sites` module and routes intact.
- Introduce a `projects` alias as the first safe transition step.
- Switch the main router import to the `projects` boundary so new code naturally grows from that seam.

## Tasks

1. Add a `projects` alias module that re-exports `sites` submodules.
2. Switch router composition in `main.rs` to use the `projects` alias.
3. Document the intent in milestone and phase artifacts.

## Success Criteria

- The backend compiles with `projects` as the architectural boundary.
- Runtime behavior remains unchanged.
- Future planning can target `projects` instead of `sites`.
