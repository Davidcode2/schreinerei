## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing (space-y-2) |
| md | 16px | Default element spacing (space-y-4, py-4) |
| lg | 24px | Section padding (space-y-6) |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions: 
- **Touch target minimum: 44x44px** for all interactive elements on mobile
- Dot indicator spacing: 6px between dots (not multiple of 4, but standard for pagination indicators)

Accent reserved for:

### Visual Hierarchy

**Primary visual anchor:** Step indicator draws the eye first (top center, shows progress and context).

**Secondary anchors (in order):**
1. Active form inputs and required field indicators
2. Primary CTA button (bottom of modal)
3. Navigation buttons (Back/Continue)

This hierarchy guides the user through the step wizard flow: orient → input → action.

## Mobile Modal Interaction Contract

### Touch Target Requirements

| Element | Minimum Size | Current | Required Change |
|---------|--------------|---------|-----------------|
| Close button | 44x44px | 16x16px | Increase to `h-11 w-11` with centered icon |
| Step navigation buttons | 44x44px | N/A | Use `h-11 min-w-[44px]` |
| Dot indicators | 24x24px tap area | N/A | Each dot 8px with 16px touch padding |

### Close Button Specification

```tsx
// Before (current)
<DialogPrimitive.Close className="absolute right-4 top-4 rounded-sm opacity-70 ...">
  <X className="h-4 w-4" />
</DialogPrimitive.Close>

// After (required)
<DialogPrimitive.Close className="absolute right-3 top-3 rounded-lg p-2.5 opacity-70 ring-offset-background transition-all hover:opacity-100 hover:bg-accent focus:outline-none focus:ring-2 focus:ring-ring focus:ring-offset-2 disabled:pointer-events-none">
  <X className="h-5 w-5" />
  <span className="sr-only">Schließen</span>
</DialogPrimitive.Close>
```

### Step Indicator Specification

- **Container**: `flex items-center justify-center gap-1.5 py-3`
- **Dot (inactive)**: `h-2 w-2 rounded-full bg-muted-foreground/30 transition-all`
- **Dot (active)**: `h-2 w-2 rounded-full bg-primary transition-all`
- **Dot (completed)**: `h-2 w-2 rounded-full bg-primary transition-all`
- **Touch target**: Each dot wrapped in a 44x44px tap area

### Swipe Navigation Contract

- **Swipe threshold**: 50px minimum drag distance
- **Swipe direction**: Horizontal only (left/right)
- **Animation**: 200ms ease-out transition between steps
- **Gesture indicator**: Subtle swipe hint on first view (optional)
- **Prevent body scroll**: During swipe gesture

### Step Transition Animation

```css
/* Step enter/exit */
.step-enter {
  opacity: 0;
  transform: translateX(100%);
}
.step-enter-active {
  opacity: 1;
  transform: translateX(0);
  transition: all 200ms ease-out;
}
.step-exit {
  opacity: 1;
  transform: translateX(0);
}
.step-exit-active {
  opacity: 0;
  transform: translateX(-100%);
  transition: all 200ms ease-out;
}
```

## Modal Height Constraints

| Screen Height | Behavior |
|---------------|----------|
| < 667px (iPhone SE) | Max 85vh, scroll within step content |
| 667px - 812px (iPhone 8-X) | Max 90vh, scroll within step content |
| > 812px | Max 90vh, natural height |

**Scroll behavior**: 
- Each step scrolls independently
- Header and footer (step indicator + buttons) remain fixed
- Only the form content area scrolls
