<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Router URL → viewer selection | Untrusted route params select a media item inside the authenticated site detail page |
| Protected blob fetch → browser download/preview | Authenticated binary content is transformed into object URLs and downloaded locally |
| Clipboard write → share flow | App-generated deep links leave the application via the user clipboard |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-36-04 | S | route param selection | mitigate | Resolve viewer target only from already-fetched tenant-scoped site activities; unmatched params show the inline viewer error panel instead of guessing another attachment. |
| T-36-05 | I | share/download actions | mitigate | Share only app routes, never storage keys or unauthenticated blob endpoints; download via authenticated `apiClient.getBlob`. |
| T-36-06 | D | fullscreen viewer | accept | Blob preview/download is user-initiated and scoped to one selected media item; no background prefetch loop is introduced. |
</threat_model>
