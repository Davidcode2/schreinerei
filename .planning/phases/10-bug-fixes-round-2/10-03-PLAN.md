Fix BUG-003: Wrong API URL (service worker intercepting localhost).

In `frontend/vite.config.ts`:

Update the runtimeCaching configuration for API calls (lines 47-62). Change the urlPattern to EXCLUDE localhost:

```typescript
// API calls - NetworkFirst with offline fallback
{
  // Match API URLs but exclude localhost (let Vite proxy handle them)
  urlPattern: /^https:\/\/(?!localhost)[^\/]+\/api\/v1\/.*/i,
  handler: 'NetworkFirst',
  options: {
    cacheName: 'api-cache',
    networkTimeoutSeconds: 10,
    expiration: {
      maxEntries: 100,
      maxAgeSeconds: 60 * 60,
    },
    cacheableResponse: {
      statuses: [0, 200]
    }
  },
},
```

The key change is the regex pattern:
- BEFORE: `/^https:\/\/.*\/api\/v1\/.*/i` — matches ALL https URLs including localhost
- AFTER: `/^https:\/\/(?!localhost)[^\/]+\/api\/v1\/.*/i` — excludes localhost

This uses a negative lookahead `(?!localhost)` to ensure localhost URLs are not matched by the service worker. The Vite dev server proxy will handle localhost API calls instead.

Note: In production, the API URL will be different (not localhost), so this pattern will correctly cache production API calls while letting dev proxy work.
