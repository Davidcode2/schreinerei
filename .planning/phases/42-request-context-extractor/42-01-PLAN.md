# Phase 42 Plan

## Goal

Remove manual request-to-tenant-context conversion from API handlers.

## Decisions

- Keep auth middleware responsible for authentication.
- Make `TenantContext` itself extractable from request extensions.
- Refactor route handlers to receive `TenantContext` directly.

## Tasks

1. Implement `FromRequestParts` for `TenantContext`.
2. Add IAM helpers that can bootstrap users from request context.
3. Refactor IAM, Inventory, Fleet, and Sites route modules to use `TenantContext` directly.
4. Verify the backend test suite still passes.

## Success Criteria

- No route handlers rebuild `TenantContext` manually from auth.
- Request-scoped tenant context remains tenant-safe.
- Existing behavior remains intact under test.
