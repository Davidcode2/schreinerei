<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → timeline composer | Untrusted note text and file metadata enter the creation flow |
| timeline composer → sites API | Client uploads must stay within allowed MIME/size constraints and existing tenant-scoped endpoints |
| offline queue → reconnect sync | Deferred camera submissions must not create duplicate or malformed activities |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-45-01 | Tampering | `ProjectTimelineComposer` file selection | mitigate | Keep the existing allowlist (`image/jpeg`, `image/png`, `image/webp`, `application/pdf`) and 10 MB limit in the shared composer before any upload request is sent. |
| T-45-02 | Denial of service | multi-file upload UI | mitigate | Reject oversize files individually, keep submission disabled during pending mutations, and avoid auto-upload loops by sending files only on explicit submit. |
| T-45-03 | Repudiation | camera/offline convergence path | mitigate | Preserve the explicit offline queue path only for the camera wrapper, so reconnect replay still creates one intentional activity per queued capture rather than ambiguous partial uploads. |
| T-45-04 | Information disclosure | attachment preview state | mitigate | Reuse existing authenticated attachment endpoints and do not introduce direct storage-key or unauthenticated preview URLs in the shared composer. |
</threat_model>
