## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| mobile browser → composer state | User-controlled files and note text enter the unified timeline submission flow |
| composer → sites API | Attachment uploads and activity creation must preserve the existing tenant-scoped, protected-media contract |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-45-01 | Tampering | `ProjectTimelineComposer` file intake | mitigate | Keep the current MIME allowlist and 10 MB size guard in the shared composer before any upload mutation fires. |
| T-45-02 | Information disclosure | attachment-backed media flow | mitigate | Reuse `useUploadSiteAttachment` and protected attachment URLs; do not switch unified submissions to direct object-storage URLs or public image paths. |
| T-45-03 | Repudiation | unified activity creation | mitigate | Create exactly one activity per submission using existing `create_activity` semantics so audit/event flow remains tied to the normal activity record. |
| T-45-04 | Denial of service | mobile camera shortcut | mitigate | Keep camera-first behavior lightweight by preloading one captured file into the shared composer instead of forcing a heavy multi-step desktop-style wizard. |
</threat_model>
