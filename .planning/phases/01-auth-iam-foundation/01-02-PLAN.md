<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → API | Untrusted input from external clients |
| API → Database | Trusted data store with tenant isolation |
| User A → User B (different tenant) | Never allowed to cross |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Elevation | Role update | mitigate | Only admins can update roles, validated in service layer |
| T-02-02 | Info Disclosure | User list | mitigate | Only admins can list users, tenant-scoped results |
| T-02-03 | Elevation | Cross-tenant access | mitigate | All queries filtered by tenant_id from JWT, not request body |
| T-02-04 | Spoofing | User invitation | accept | V1 uses admin-initiated invite only, no self-registration |
| T-02-05 | Tampering | Profile update | mitigate | Users can only update own profile, validated in service |
</threat_model>
