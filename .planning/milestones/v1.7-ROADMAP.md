# Milestone v1.7: Active Project Context + Gap Closure

**Status:** ✅ SHIPPED 2026-05-01

## Overview

Enable automatic assignment of materials/tools to the currently active construction site, with full end-to-end verification and gap closure for audit findings.

## Phases

**Goal**: Backend stores and validates user's active Baustelle preference
**Depends on**: Phase 21 (v1.6 complete)
**Plans**: 4 plans

Plans:

- [x] 22-01: UserPreferences Repository & Service with validation

## Milestone Summary

**Key Decisions:**

- Active Baustelle is user-scoped (not global) — teams work different sites
- Deterministic hash-based colors (no manual selection) — simpler UX
- JSONB storage for preferences — flexible schema evolution
- site_id is optional in WithdrawMaterial — backward compatible
- FK-safe tenant-local user mapping — prevents 500 errors on preferences API

**Issues Resolved:**

- Missing phase verification artifacts for phases 22/23
- DEDU-03 integration gap (backend producer, no frontend consumer)
- Preferences API FK violations when Keycloak subject used directly as local user FK

**Technical Debt Incurred:**

- Preferences cache invalidation after site delete could be tighter (minor staleness window)
- Offline withdraw queue omits site_id (online/offline behavior mismatch)
- No Nyquist validation artifacts for phases 22-25

**Deferred to v2.0:**

- Active preference in IndexedDB for offline access
- Preference sync when connectivity restored
- Stale active project handling during offline
- Dashboard filter by active Baustelle
- Dashboard active Baustelle statistics
