# Requirements: Schreinerei v1.10 Baustelle Activity Stream Features

**Archived:** 2026-05-01
**Core Value:** Mitarbeiter finden alles schnell, Chefs haben den Überblick. Weniger Suchzeit, weniger Fehler.

## v1.10 Requirements

### Camera Upload (CAM)

- [x] **CAM-01**: Camera button on activity stream opens native camera/gallery picker (not document modal) — ✅ Delivered
- [x] **CAM-02**: User can add an optional text note when uploading via camera flow — ✅ Delivered
- [x] **CAM-03**: Camera upload automatically attaches selected image to activity entry — ✅ Delivered

### Document Upload (DOC)

- [x] **DOC-01**: Document modal supports note AND attachment(s) in a single entry — ✅ Delivered
- [x] **DOC-02**: User can upload one or more PDFs as attachments (with optional note) — ✅ Delivered
- [x] **DOC-03**: User can upload one or more images as attachments (with optional note) — ✅ Delivered
- [x] **DOC-04**: User can create an entry with attachment-only (no note text required) — ✅ Delivered
- [x] **DOC-05**: Document modal accepts both PDF and image file types with appropriate validation — ✅ Delivered

### Media Viewer (VIEW)

- [x] **VIEW-01**: Click on image preview in feed opens fullscreen modal showing large image — ✅ Delivered
- [x] **VIEW-02**: Click on document preview in feed opens fullscreen modal showing document — ✅ Delivered
- [x] **VIEW-03**: Fullscreen modal has its own slug-based URL for direct linking and sharing — ✅ Delivered
- [x] **VIEW-04**: Fullscreen modal displays note text alongside the media — ✅ Delivered
- [x] **VIEW-05**: Fullscreen modal shows timestamp and username of entry creator — ✅ Delivered
- [x] **VIEW-06**: Fullscreen modal has download button to download the media file — ✅ Delivered
- [x] **VIEW-07**: Fullscreen modal has share button to copy page link to clipboard — ✅ Delivered
- [x] **VIEW-08**: Fullscreen modal has close button to return to activity feed — ✅ Delivered
- [x] **VIEW-09**: Modal fills almost the entire screen for optimal media viewing — ✅ Delivered

### Entry Management (ENTRY)

- [x] **ENTRY-01**: User can delete their own activity entries (creator-only permission) — ✅ Delivered
- [x] **ENTRY-02**: Delete action shows confirmation dialog before removing entry — ✅ Delivered
- [x] **ENTRY-03**: Deletion removes entry and associated attachments from feed — ✅ Delivered

## Outcome Notes

- All 20 milestone requirements were delivered and verified.
- Human UAT was completed for Phase 32 and Phase 33.
- Security verification was completed for Phase 32 and Phase 33.
- Two post-execution correctness fixes were required during verification: optional thumbnail handling for non-image attachments and creator-name hydration after photo activity creation. Both are now incorporated into the shipped milestone state.
