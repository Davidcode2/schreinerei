## Milestone Summary

**Key Decisions:**

- Rust Backend with Axum 0.8, SQLx 0.8
- DDD layering: domain/application/infrastructure/api
- Multi-tenant via TenantId in all queries
- Keycloak OAuth2 PKCE for SPA auth
- IndexedDB (Dexie.js) for offline storage
- Keycloak Organizations for tenant isolation (v1.1)
- User ID resolution: Keycloak ID → local users.id for FK references
- WorkType enum: site, workshop, travel, other

**Issues Resolved:**

- FK constraint violations (stock_entries, reservations, time_entries)
- NUMERIC vs FLOAT8 type mismatches
- WorkType enum mismatch between frontend/backend
- Nullable user_name in reservation details
- Calendar date format parsing
- Token exchange/refresh cascade failures
- Fleet button non-functional
- User management loading state stuck
