# Milestone v1.9: Inventory Features

**Status:** ✅ SHIPPED 2026-05-01
**Phases:** 30-33
**Total Plans:** 13

## Overview

Made inventory fully manageable end to end: categories can be managed from settings, materials can be edited directly, stock can be added or corrected, and enriched history now gives users the context needed to trust what happened.

## Milestone Summary

**Key Decisions:**
- Inventory settings live at `/settings/inventory` instead of a generic settings fallback.
- Stock-in is a dedicated command and UI flow, separate from stock adjustment.
- Category deletion preserves history and stays blocked while referenced materials exist.
- Enriched inventory history is its own frontend query/presenter path.
- Inventory DTO drift is controlled through generated bindings with a thin local facade.
- Milestone close accepted the remaining Phase 33 history color/assertion gap through explicit manual verification instead of blocking shipment.

**Issues Resolved:**
- Missing category-management surface in the UI
- No direct material editing or stock-in path in inventory details
- Inventory history lacked user attribution and site navigation context
- Remaining handwritten order DTO drift in the frontend facade

**Issues Deferred:**
- Metadata-only history audit entries for `location_changed` and `min_quantity_changed`
- Historical non-v1.9 planning artifacts acknowledged at milestone close

**Technical Debt Incurred:**
- Phase 33's final history badge color/assertion hardening work was accepted manually and should return as explicit future quality work if still desired
